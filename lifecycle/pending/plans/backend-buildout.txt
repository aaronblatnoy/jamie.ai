## 0. CONTEXT & GOALS

Problem statement
-----------------
jamie.ai is an AI-powered mock investment-banking interviewer. The
frontend (separate, not in scope) asks questions out loud, transcribes
the user's spoken answer in the browser (Web Speech API), and needs a
backend to (a) serve a random unseen question from a category filter,
(b) score the transcribed answer against the question's key_points using
Claude, and (c) turn text into speech via ElevenLabs. The backend is a
thin, stateless proxy — session state lives entirely on the client.

Success criteria (measurable)
-----------------------------
- `npm run dev` boots an Express server on PORT (default 3001) with no
  unhandled-rejection or import errors.
- `GET /api/question?mode=ib` returns a single JSON question object with
  exactly the keys { id, question, key_points, category, difficulty } and
  HTTP 200; never leaks model_answer.
- `GET /api/question?mode=ib&categories=basic/accounting&difficulty=1&exclude=mi-accounting-basic-001`
  returns a difficulty-1 basic/accounting question whose id is not the
  excluded one; returns `{ done: true }` (HTTP 200) when the filtered,
  de-excluded pool is empty.
- `POST /api/evaluate` with a valid body returns
  `{ hit: string[], missed: string[], feedback_line: string, score: number }`
  with score in [0,100], parsed from Claude's structured output, HTTP 200.
- `POST /api/tts` with `{ text }` streams `Content-Type: audio/mpeg` bytes
  back; a bad/missing ElevenLabs key yields a clean 5xx JSON error, not a
  crash.
- No endpoint ever returns ANTHROPIC_API_KEY or ELEVENLABS_API_KEY, and
  neither is logged.

Non-goals (explicitly out of scope)
-----------------------------------
- Frontend (React/Vite), STT, audio playback, session/score persistence.
- Auth / rate limiting / multi-tenant concerns (single-user local tool).
- A database. Questions are read from JSON on disk on demand.
- Guide parsing / PDF ingestion (questions are already generated).
- Deployment / Docker / CI.
- A "behavioral" mode (no behavioral data exists on disk; only ib/rx).

Assumptions made (VERIFIED against the repo unless noted)
---------------------------------------------------------
A1. Data path mapping. The frontend passes mode=ib|rx, but IB data lives on
    disk under questions/400-mi/, not questions/ib/. The backend MUST map
    mode -> directory: ib -> 400-mi, rx -> rx. (Verified:
    questions/400-mi/basic/accounting/questions.json is 45KB; questions/ib/
    does not exist.)
A2. Canonical IB category list = manifest.json. The authoritative IB slug
    set is the sections[].slug array in questions/400-mi/manifest.json (13
    sections: basic/accounting, advanced/accounting,
    basic/enterprise-equity-value, advanced/enterprise-equity-value,
    basic/valuation, advanced/valuation, basic/dcf, advanced/dcf, basic/ma,
    advanced/ma, basic/lbo, advanced/lbo, brain-teasers). The stale bare dirs
    accounting/, dcf/, valuation/, lbo/, ma/, fit/ have been deleted.
    (Verified: manifest now uses basic/<category> and advanced/<category> slugs.)
A3. RX data is not yet populated. The RX category dirs exist but contain no
    questions.json (verified: questions/rx/*/questions.json glob matches
    nothing). RX slugs (from dir names): accounting-and-financial-statements,
    bond-math-and-valuation, capital-structure-and-debt-instruments,
    chapter-11-and-plan-of-reorganization, credit-and-covenant-analysis,
    distressed-investing-and-trading, out-of-court-restructuring,
    recovery-and-waterfall-analysis, rx-overview-and-advisory. The loader
    MUST treat a missing/empty questions.json as "zero questions for that
    slug" -- never throw. RX has NO basic/advanced split and NO manifest.json,
    so RX "all categories" = enumerate the rx/ subdirectories at request time.
A4. TWO ElevenLabs voices, not one. The request text says one voice
    (ELEVENLABS_VOICE_ID), but the on-disk .env.example has
    ELEVENLABS_VOICE_ID_IB and ELEVENLABS_VOICE_ID_RX. We follow the
    .env.example (it is the source of truth). POST /api/tts accepts an
    optional voice_id override; when absent it selects by an optional mode
    in the body (ib->_IB, rx->_RX), falling back to _IB. Flagged in OPEN
    QUESTIONS.
A5. SDK is @anthropic-ai/sdk ^0.52.0 (from package.json) and is NOT yet
    installed (no node_modules). npm install is Phase 0. 0.52 predates the
    messages.parse() helper, so we use
    messages.create({ output_config: { format: { type:"json_schema", schema }}})
    and JSON.parse the first text block. (The output_format top-level param
    is deprecated; use output_config.format.)
A6. CLAUDE.md describes an older architecture (consolidated
    backend/src/data/ib-questions.json) that does NOT match the current
    on-disk reality (per-category files under questions/). We follow the
    REQUEST + actual data layout (per-category, on-demand read from
    questions/), not the stale CLAUDE.md paragraph. backend/src/data/ is
    left empty/unused. Flagged in OPEN QUESTIONS.
A7. Claude config (locked by project rule + claude-api guidance): model
    claude-opus-4-8, thinking:{type:"adaptive"}. No assistant prefill (400s
    on this model). max_tokens ~2048 is ample for the small JSON result;
    non-streaming is fine (well under the ~16K streaming threshold).

## 1. ARCHITECTURE OVERVIEW

How this fits into the existing system
--------------------------------------
This is the only backend. It sits between the React frontend and two
external APIs (Anthropic, ElevenLabs). It reads question JSON straight from
the repo's questions/ tree (sibling of backend/). No DB, no queue, no auth.

Affected services
-----------------
- api (Express) — NEW, this whole plan.
- db — none.   worker — none.   web — out of scope (consumes endpoints).
- shared — the on-disk questions/ tree is the shared data source (read only).

Data flow (per question round-trip)
-----------------------------------
  [frontend] --GET /api/question?mode&categories&difficulty&exclude-->
  [questions.js] --reads--> questions/{400-mi|rx}/{slug}/questions.json
       |  filter difficulty, drop excluded ids, pick 1 at random
       v
  {id,question,key_points,category,difficulty}  (model_answer stripped)

  [frontend] --POST /api/evaluate {question,key_points,user_answer,mode}-->
  [evaluate.js] --loads prompts/{mode}-system.txt--> [Anthropic SDK]
       |  claude-opus-4-8, adaptive thinking, output_config.format
       v
  {hit,missed,feedback_line,score}

  [frontend] --POST /api/tts {text,voice_id?,mode?}-->
  [tts.js] --fetch--> ElevenLabs stream --pipe--> audio/mpeg bytes

Invariants this plan must preserve
----------------------------------
- INV1 (KEY SECRECY): ANTHROPIC_API_KEY / ELEVENLABS_API_KEY never appear
  in any response body, header, or log line. All external calls server-side.
- INV2 (STATELESSNESS): backend holds no per-session state; exclude comes
  from the client each call.
- INV3 (NO MODEL-ANSWER LEAK): GET /api/question strips model_answer.
- INV4 (MODE->PATH MAPPING): every read goes through ib->400-mi / rx->rx;
  no route hardcodes a raw mode as a dir name.
- INV5 (ESM ONLY): import/export throughout (package.json "type":"module").
  node:fs/promises; resolve paths via fileURLToPath(import.meta.url), no
  __dirname.
- INV6 (CLAUDE CONFIG): claude-opus-4-8 + thinking:{type:"adaptive"},
  structured output via output_config.format, never a prefill.

## 5. API CONTRACT

GET /api/question
  Query: mode=ib|rx (req); categories=csv (opt); difficulty=1|2|3 (opt);
         exclude=csv ids (opt). Auth: none.
  200: { id, question, key_points: string[], category, difficulty }
       OR { done: true } when pool empty/exhausted.
  400: { error } for missing/invalid mode.

POST /api/evaluate
  Body: { question, key_points: string[], user_answer, mode:"ib"|"rx" }.
  Auth: none.
  200: { hit: string[], missed: string[], feedback_line: string,
         score: number (0-100 int) }.
  400: missing question/user_answer or bad mode.
  502: Claude upstream/parse failure (no key / raw SDK leak).

POST /api/tts
  Body: { text, voice_id?, mode?:"ib"|"rx" }. Auth: none.
  200: binary audio, Content-Type: audio/mpeg (streamed).
  400: text missing.   502: { error:"tts_failed" } (no key leak).

GET /api/health (Phase 0)  -> 200 { ok: true }.

Modified endpoints: none (greenfield).

  - P4's two prompt files are disjoint — safe true parallel.
  - questions-store reads manifest + per-category files once and caches —
